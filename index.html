Top 10 Reasons to Use RubyMine

What is RubyMine?

It's an IDE, silly!

Yes this means it's more than just an editor. IDEs help you understand development projects as a whole.

WARNING: Controversial statement ahead!

Someone who's familiar with an IDE will always be more efficient than someone (of the same skill level) who's merely using an editor.

What do you mean by "editor"?

Vim, Emacs, Sublime, Atom, etc

Full disclosure: I love Vim.

I have a lovingly maintained setup that I use for Go and one-offs. It very well may be far more customized than what you use.

Why is there IDE hate in scripting language communities?

I've found that people who've worked with compiled code a lot have less of a hard time w/ the idea of an IDE. I don't buy the argument that says they are "crutches" or only for "people who think a certain way".

The actual Top 10

So now that we know that there's basically no reason for IDE hate in the Ruby world beyond the tautological, let's talk about why you'd want to use this.

10. Control your Rails project [ADD SCREENSHOT OF CONFIGURATION MANAGEMENT]

Run specs, rake tasks, bundler commands, and more from inside RubyMine

9. Everything in one window

If you want to code on the couch, you can have all your stuff (including a terminal window) in one place. Work in the terminal, run bundler commands and rake tasks, use a Rails console, etc.

8. Full Git support

Deal with diffs, compare versions, switch branches, open lines on GitHub, create Gists from selections. Yes, many editors have plugins that can do this. W/ RubyMine, you don't have to install anything extra.

7. Context Save/Load

Save all open files and their positions as a "context" and then come back to it later. Perfect for when you have to jump on top of a hotfix but then want to get right back to all your code windows, etc later.

6. Live Indexing: Crazy Code Navigation

Cmd-Click on a method call or class name to go straight to its definition. Go "forward" and "back" in the stack of files (and their positions!) you've been looking at

5. Quickly Run Tests at Any Scope

Put the cursor on the single test or (in RSpec/MiniTest) on the describe/context block start, use a key combo to run it in normal mode or through the debugger. Combine with window pane navigation to quickly run the last test you ran.

4. Refactoring

Related: Find Usage to get a collection of all the times a piece of code is used (even in dynamic calls!). This list is of course click-to-navigate. Quickly rename methods or classes globally (even in comments!) Extract portions of code into new methods or variables, and more.

3. All Stack Traces are Hyper Links

Any stack trace --whether in a test failure, rake task, or anywhere else -- is a collection of hyperlinks into your code. This saves an AMAZING amount of time.

2. ShiftShift to find ANYTHING

Single, unified, fuzzy search for class names, file names, and even the names of methods.

1. Graphical Debugger

It's impossible to overstate how much time this has saved me over the years. Pry is great. I use it. But it's not graphical -- you have to reload your process. Effortlessly step into method calls, step over them to trace execution line-by-line, set watch expressions, break on exceptions (named and generally), and tons of other stuff. One really killer feature here: Shift-Cmd-D to run the spec under your cursor in debug mode.
